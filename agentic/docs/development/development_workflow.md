Before writing code:

1. Read the architecture documents.
2. Read any ADRs related to the capability.
3. Check whether the functionality already exists.
4. Reuse existing abstractions.

When coding:

- Make the smallest coherent change.
- Don't duplicate code.
- Update tests.
- Keep commits focused.
- Explain architectural decisions.

After coding:

- Run tests.
- Fix lint.
- Update documentation.
- Update ADRs if architecture changed.
